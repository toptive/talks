REST vs GraphQL

Para ilustrar las principales diferencias entre REST y GraphQL cuando se trata de obtener datos de una API, consideremos un ejemplo simple: en una aplicación de blogging, se necesita mostrar los títulos de las publicaciones de un usuario específico. La misma pantalla también muestra los nombres de los últimos 3 seguidores de ese usuario.

¿Cómo se resolvería esa situación con REST y GraphQL?

With REST

Notemos que estamos haciendo overfetching dado que los endpoints retornan información adicional que no necesitamos.

With GraphQL

Usando GraphQL, el cliente puede especificar exactamente los datos que necesita en una consulta.

No more Over- and Underfetching

Uno de los problemas más comunes con REST es el over/underfetching. Esto sucede porque la única forma en que un cliente puede descargar datos es utilizando los endpoints que devuelven estructuras de datos fijas. Es muy difícil diseñar la API de manera que pueda proporcionar a los clientes los datos exactos que necesitan.

Benefits of a Schema & Type System

GraphQL usa un sistema fuertemente tipado para definir las capacidades de una API. Todos los tipos que se exponen en una API se escriben en un esquema utilizando el lenguaje de definición de esquema GraphQL (SDL). Este esquema sirve como el contrato entre el cliente y el servidor para definir cómo un cliente puede acceder a los datos.
